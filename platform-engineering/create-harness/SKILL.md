# Create Harness — Claude Code Agent Harness Scaffolder

You are a Platform Engineer bootstrapping a **harness**: a versioned monorepo that holds a team's canonical agent configuration (skills, MCP servers, sub-agents, a knowledge vault, and a workspace of product repos) and *projects* it into Claude Code via symlinks generated by `bin/harness sync`. This skill builds a **Claude-Code-only** harness — it deliberately omits Devin and any other CLI adapter.

## Overview

**Core principle:** a harness keeps **one canonical source** of skills/MCP/agents/knowledge and generates the `.claude/` projection from it — you never hand-edit the projection. This scaffolder produces that source tree, wires the contemplated repositories into `workspace/`, and builds the graphify knowledge graph so the agent can answer architectural questions from the graph instead of re-reading source.

A finished harness looks like this (Claude-only):

```text
<harness>/
  AGENTS.md                 # the single agent contract (CLAUDE.md is a symlink to it)
  CLAUDE.md -> AGENTS.md
  harness.config.yaml       # active profile: tools (claude only), active_skills, active_agents
  bin/
    harness                 # the CLI: sync | doctor | skills | graph | new-project
    render_mcp.py           # mcp/servers.json -> .mcp.json
    render_plugins.py       # catalog/plugins/plugins.json -> .claude/settings.json
  catalog/
    skills/                 # library of skills (only active ones get symlinked)
    plugins/plugins.json    # marketplace + enabled plugins (projected to .claude/settings.json)
  agents/                   # sub-agent profiles (<name>/AGENT.md)
  mcp/servers.json          # canonical MCP server list (auth is local per dev)
  vault/                    # knowledge: team/ (editable) + optional inteligencia/ submodule
    index.md
    team/
  workspace/
    workspace.yaml          # manifest of contemplated repos
    <repo>/ ...             # cloned product repos
  graphify-out/             # federated knowledge graph (generated by `harness graph`)
  .gitignore
  .graphifyignore
```

## When to use

- Standing up a brand-new team/project harness for Claude Code.
- Migrating an ad-hoc collection of skills/MCP into a canonical, projected layout.
- The user wants the harness wired to specific repositories AND a graphify graph across them.

## When NOT to use

- The harness already exists and you only need to add a repo → run `./bin/harness new-project <name> [url]` then `./bin/harness graph`.
- You only need to (re)generate the projection → run `./bin/harness sync`.
- The user needs a Devin / multi-CLI harness → this skill is Claude-only; adapt the topology with a Devin adapter and a Devin branch in `bin/harness sync` instead.

## Step 0 — Interview the operator (REQUIRED before any file is written)

Do not scaffold blindly. Ask the operator the questions below and wait for answers. The **repository list is mandatory** — it drives both `workspace/workspace.yaml` and the graphify step.

1. **Harness location & name.** Where should the harness root be created (absolute path)? What `name:` goes in `harness.config.yaml`?
2. **Contemplated repositories (the key question).** For each repo to include, collect:
   - `name` (the workspace directory name)
   - `url` (git clone URL — required so the skill can clone it and graphify can run)
   - `domain` (e.g. core, identity, frontend, infra, ci)
   - `type` (service | frontend | iac | helm-chart | github-workflows | library)
   - `summary` (one line: what the repo does)
   Loop until the operator says "done". If a repo has no remote yet, record it and skip cloning that one (graphify will skip it with a warning).
3. **MCP servers.** Which of linear / github / context7 / figma to enable (default: linear, context7, github enabled; figma off)? Any custom server?
4. **Plugins.** Enable the `oh-my-claudecode` marketplace plugin? (default: yes — multi-agent orchestration.)
5. **Vault knowledge.** Include a read-only macro-knowledge **git submodule** under `vault/inteligencia/` (provide URL), or start with an empty `vault/team/` only?
6. **Active skills.** Any skills to pre-activate in `catalog/skills/` now, or start empty (recommended — keep the active set small to protect the context window)?

Echo the collected answers back as a short plan and get a go-ahead before writing.

## Step 1 — Scaffold the topology

Create the directory tree and write the canonical files. Use the templates below verbatim (substitute the interview answers). All paths are relative to the harness root.

### 1a. Directories

```bash
HARNESS="<absolute-path-from-interview>"
mkdir -p "$HARNESS"/{bin,catalog/skills,catalog/plugins,agents,mcp,vault/team,workspace}
cd "$HARNESS"
git init -q 2>/dev/null || true
```

### 1b. `harness.config.yaml` (Claude-only)

```yaml
# Active harness profile. `harness sync` reads from here.
name: <harness-name>

tools:
  - claude          # Claude Code only — no devin adapter

active_skills:
  # Keep this small and task-scoped to protect the context window.
  # Add skills present under catalog/skills/ here to symlink them into .claude/skills.

active_agents:
  # Sub-agent profiles under agents/<name>/AGENT.md. Activate when a concrete flow needs one.

vault:
  team: vault/team
  # inteligencia: vault/inteligencia   # uncomment if you add the submodule
```

### 1c. `AGENTS.md` + `CLAUDE.md` symlink

Write a contract that documents the topology and the pre/post-execution routine (clone repos → load vault → query graph → plan → execute with a human gate before PR; after code changes, rebuild graphify). Then make `CLAUDE.md` a symlink so Claude Code reads the same file:

```bash
ln -sfn AGENTS.md CLAUDE.md
```

### 1d. `bin/harness` (Claude-only CLI)

```bash
#!/usr/bin/env bash
# harness — projects the canonical harness content into Claude Code (.claude/, .mcp.json).
set -euo pipefail
ROOT="$(cd "$(dirname "$0")/.." && pwd)"
CFG="$ROOT/harness.config.yaml"
cd "$ROOT"

# extract items of a simple YAML list under a top-level key
yaml_list() { # $1=key $2=file
  awk -v k="$1" '
    $0 ~ "^"k":[[:space:]]*$" {f=1; next}
    f && /^[[:space:]]*-[[:space:]]/ {sub(/^[[:space:]]*-[[:space:]]*/,""); sub(/[[:space:]]*#.*/,""); gsub(/"/,""); print; next}
    f && /^[^[:space:]#]/ {f=0}
  ' "$2"
}

cmd_sync() {
  local skills agents
  skills=$(yaml_list active_skills "$CFG")
  agents=$(yaml_list active_agents "$CFG")
  ln -sfn AGENTS.md CLAUDE.md
  echo "  CLAUDE.md -> AGENTS.md"
  mkdir -p .claude/skills .claude/agents
  for s in $skills; do ln -sfn "../../catalog/skills/$s" ".claude/skills/$s"; done
  for a in $agents; do ln -sfn "../../agents/$a/AGENT.md" ".claude/agents/$a.md"; done
  python3 bin/render_mcp.py mcp/servers.json claude > .mcp.json
  if [ -f catalog/plugins/plugins.json ]; then
    python3 bin/render_plugins.py catalog/plugins/plugins.json mcp/servers.json > .claude/settings.json
    echo "  claude:  .claude/skills/* .claude/agents/* .mcp.json .claude/settings.json"
  else
    echo "  claude:  .claude/skills/* .claude/agents/* .mcp.json"
  fi
  echo "sync ok."
}

cmd_doctor() {
  local skills agents ok=0
  skills=$(yaml_list active_skills "$CFG")
  agents=$(yaml_list active_agents "$CFG")
  echo "== binaries =="
  command -v claude >/dev/null && echo "  claude: ok" || echo "  claude: missing (npm i -g @anthropic-ai/claude-code)"
  command -v graphify >/dev/null && echo "  graphify: ok" || echo "  graphify: missing (uv tool install graphifyy)"
  echo "== active skills =="
  for s in $skills; do
    if [ -f "catalog/skills/$s/SKILL.md" ]; then echo "  $s: ok"; else echo "  $s: SKILL.md MISSING"; ok=1; fi
  done
  echo "== sub-agents =="
  for a in $agents; do
    if [ -f "agents/$a/AGENT.md" ]; then echo "  $a: ok"; else echo "  $a: AGENT.md MISSING"; ok=1; fi
  done
  echo "== MCP =="
  python3 - "mcp/servers.json" << 'PY'
import json,sys
d=json.load(open(sys.argv[1]))
for n,s in d.get("servers",{}).items():
    st="ENABLED" if s.get("enabled") else "off"
    print(f"  {n}: {st} ({s.get('transport','stdio')}) — auth: {s.get('auth','-')}")
print("  reminder: credentials are local (env / *.local.json), never in the repo.")
PY
  return $ok
}

cmd_skills() {
  echo "== catalog =="; ls -1 catalog/skills 2>/dev/null || true
  echo "== active =="; yaml_list active_skills "$CFG"
}

cmd_graph() {
  command -v graphify >/dev/null || { echo "graphify missing (uv tool install graphifyy)"; exit 1; }
  # 1) build/refresh the graph of EVERY cloned repo (local, no LLM).
  #    `graphify update .` CREATES graphify-out/graph.json on first run, so this also
  #    graphs freshly-cloned repos that have no committed graph yet — do NOT gate on a
  #    pre-existing graph or new clones get silently skipped.
  for repo in workspace/*/; do
    [ -d "$repo" ] || continue
    [ -e "$repo/.git" ] || continue   # only real clones, not stray dirs
    echo "  update: $repo"
    (cd "$repo" && graphify update . >/dev/null 2>&1) || echo "    (failed — run manually)"
  done
  # 2) the harness's own graph (vault/ catalog/ agents/ bin/ — workspace/ excluded via .graphifyignore)
  echo "  update: . (harness)"
  graphify update . >/dev/null 2>&1 || echo "    (harness update failed — federated may be stale)"
  # 3) federated: harness + repos in one cross-repo queryable graph
  set --
  [ -f graphify-out/graph.json ] && set -- "$@" graphify-out/graph.json
  for g in workspace/*/graphify-out/graph.json; do [ -f "$g" ] && set -- "$@" "$g"; done
  [ "$#" -ge 2 ] || { echo "only $# graph(s) found — clone repos and run graphify in them first"; exit 1; }
  echo "  merge:  $# graphs -> graphify-out/federated-graph.json"
  graphify merge-graphs "$@" --out graphify-out/federated-graph.json
  echo "graph ok. query: graphify query \"...\" --graph graphify-out/federated-graph.json"
}

cmd_new_project() {
  local name="${1:-}" url="${2:-}"
  [ -n "$name" ] || { echo "usage: harness new-project <name> [git-url]"; exit 1; }
  {
    echo "  - name: $name"
    [ -n "$url" ] && echo "    url: $url"
    echo "    domain: TODO"
    echo "    type: service"
    echo "    summary: \"TODO: what this repo does\""
  } >> workspace/workspace.yaml
  echo "added to workspace.yaml: $name"
  if [ -n "$url" ]; then (cd workspace && git clone "$url" "$name"); fi
}

case "${1:-help}" in
  sync)        cmd_sync ;;
  doctor)      cmd_doctor ;;
  skills)      cmd_skills ;;
  graph)       cmd_graph ;;
  new-project) shift; cmd_new_project "$@" ;;
  *) cat <<USAGE
harness — Claude Code harness tooling
  sync                      project skills/agents/mcp into Claude Code (.claude/, .mcp.json)
  doctor                    check binaries, skills, sub-agents, MCP
  skills                    list catalog vs active
  graph                     update graphify graphs of all repos + the federated root graph
  new-project <name> [url]  add a repo to workspace.yaml (and clone, if url given)
USAGE
  ;;
esac
```

Make it executable: `chmod +x bin/harness`.

### 1e. `bin/render_mcp.py`

```python
#!/usr/bin/env python3
"""Render mcp/servers.json into Claude Code's .mcp.json format.
Usage: render_mcp.py <servers.json> claude
Emits { "mcpServers": { ... } } on stdout, only enabled servers."""
import json, sys

def main():
    path = sys.argv[1]
    with open(path) as f:
        data = json.load(f)
    out = {"mcpServers": {}}
    for name, s in data.get("servers", {}).items():
        if not s.get("enabled", False):
            continue
        transport = s.get("transport", "stdio")
        if transport == "stdio":
            entry = {"type": "stdio", "command": s["command"], "args": s.get("args", [])}
            if s.get("env"):
                entry["env"] = s["env"]
        else:  # http / streamable http
            entry = {"type": "http", "url": s["url"]}
        out["mcpServers"][name] = entry
    json.dump(out, sys.stdout, indent=2, ensure_ascii=False)
    sys.stdout.write("\n")

if __name__ == "__main__":
    main()
```

### 1f. `bin/render_plugins.py`

```python
#!/usr/bin/env python3
"""Render Claude Code config (.claude/settings.json) from the catalog.
Usage: render_plugins.py <catalog/plugins/plugins.json> [mcp/servers.json]
Emits { "extraKnownMarketplaces": {...}, "enabledPlugins": {...}, "enabledMcpjsonServers": [...] }."""
import json, sys

def main():
    with open(sys.argv[1]) as f:
        data = json.load(f)
    out = {"extraKnownMarketplaces": {}, "enabledPlugins": {}}
    for name, spec in data.get("marketplaces", {}).items():
        out["extraKnownMarketplaces"][name] = {
            "source": {"source": spec["source"], "repo": spec["repo"]}
        }
    for key, spec in data.get("plugins", {}).items():
        out["enabledPlugins"][key] = bool(spec.get("enabled", False))
    # pre-approve project-scope MCP servers (only enabled) to avoid the manual "trust" prompt
    if len(sys.argv) > 2:
        with open(sys.argv[2]) as f:
            servers = json.load(f).get("servers", {})
        enabled = [n for n, s in servers.items() if s.get("enabled", False)]
        if enabled:
            out["enabledMcpjsonServers"] = enabled
    json.dump(out, sys.stdout, indent=2, ensure_ascii=False)
    sys.stdout.write("\n")

if __name__ == "__main__":
    main()
```

## Step 2 — Clone the contemplated repositories

For every repo collected in Step 0 that has a URL, clone it into `workspace/` (idempotent: skip existing, fast-forward pull). Repos without a remote are recorded only and skipped here (graphify will skip them too, with a warning).

```bash
cd "$HARNESS/workspace"
# repeat per repo:
if [ -d "<name>/.git" ]; then (cd "<name>" && git pull --ff-only); else git clone "<url>" "<name>"; fi
cd "$HARNESS"
```

## Step 3 — Project into Claude Code

```bash
cd "$HARNESS"
./bin/harness sync     # generates CLAUDE.md, .claude/skills/*, .claude/agents/*, .mcp.json, .claude/settings.json
./bin/harness doctor   # confirms binaries, active skills/agents, MCP — fix anything reported MISSING
```

## Step 4 — Update ALL graphify graphs (per-repo + federated)

This fulfills the requirement that creating the harness updates every repo's graph. Prerequisites:

- `graphify` on PATH (`uv tool install graphifyy`).
- `ANTHROPIC_API_KEY` exported in the env if you want semantic doc extraction (code graphing via tree-sitter is free and local; never put the key in the repo).
- Install the graphify git hooks per clone so commits keep graphs fresh and the merge driver avoids conflict markers in `graph.json`:

```bash
for d in workspace/*/; do (cd "$d" && graphify hook install 2>/dev/null || true); done
graphify hook install 2>/dev/null || true
```

Then build/refresh every graph and the federated root graph in one shot:

```bash
./bin/harness graph
```

`harness graph` (1) runs `graphify update .` in every cloned repo — which **creates** `graphify-out/graph.json` on first run, so newly-cloned repos get graphed too — (2) updates the harness's own graph, and (3) merges them all into `graphify-out/federated-graph.json`. It requires at least 2 graphs (≥1 cloned repo + the harness) — if it complains, confirm the repos cloned in Step 2. Verify with a query:

```bash
graphify query "list the repos in this harness" --graph graphify-out/federated-graph.json
```

Commit the shareable graph artifacts (`graph.json`, `federated-graph.json`, `GRAPH_REPORT.md`) with the harness; the machine-local ones (`cache/`, `manifest.json`, `.graphify_root`) are already gitignored.

## Verification checklist (evidence before declaring done)

Run each and confirm the output — do not claim success without it:

- [ ] `./bin/harness doctor` → claude ok, graphify ok, no MISSING skills/agents.
- [ ] `test -L CLAUDE.md && readlink CLAUDE.md` → `AGENTS.md` (symlink, not a copy).
- [ ] `cat .mcp.json` → only the enabled MCP servers appear.
- [ ] `cat .claude/settings.json` → `extraKnownMarketplaces` + `enabledPlugins` present.
- [ ] `ls workspace/` → every contemplated repo (with a URL) is cloned.
- [ ] `ls graphify-out/federated-graph.json` exists and `graphify query ... --graph graphify-out/federated-graph.json` returns results.
- [ ] Re-running `./bin/harness sync` is idempotent (no errors, no duplicate symlinks).

## Anti-patterns

| Anti-pattern | Why it's wrong | Do instead |
|---|---|---|
| Hand-editing `.claude/` or `.mcp.json` | They are generated projections — edits are lost on next `sync` | Edit the canonical source (`catalog/`, `mcp/`, `harness.config.yaml`) then `./bin/harness sync` |
| Copying `CLAUDE.md` instead of symlinking | Drifts from `AGENTS.md`; two sources of truth | `ln -sfn AGENTS.md CLAUDE.md` |
| Activating every skill in `active_skills` | Floods the agent's context window | Keep the active set small and task-scoped; the catalog can be large |
| Committing secrets / PII into `mcp/servers.json` or the vault | Leaks credentials and customer data | Use `${ENV_VAR}` placeholders and `*.local.json` (gitignored); vault holds rules/architecture, not customer data |
| Running `harness graph` before cloning repos | Needs ≥2 graphs; fails or produces a stale federated graph | Clone in Step 2 first, then `./bin/harness graph` |
| Putting the `ANTHROPIC_API_KEY` in the repo for graphify | Secret leak | Export it in the env only |
